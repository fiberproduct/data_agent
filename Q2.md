# Q2：Agent 系统设计文档

## 1. 任务目标
本项目目标是基于 `data.pq` 中的特征预测目标列 `Y1`，并通过多模型训练与评估选择更优方案，输出可复现的完整实验流程与结果。

## 2. 架构图
```mermaid
flowchart LR
    A[read_data] --> B[agent_a_plan]
    B --> C[agent_a_preprocess]
    C --> D[agent_b_train]
    D --> E[agent_c_evaluate]
```

## 3. 工作流说明
1. `read_data`
   读取 parquet、识别 `X*` 特征与目标列、产出数据概览与缺失分布图。
2. `agent_a_plan`
   使用 LLM Planner（DeepSeek）或 Rule Planner 生成清洗计划，包含：
   `split / categorical / impute / scaling / feature_select / pca`。
3. `agent_a_preprocess`
   执行计划驱动的预处理，重点是时序防泄漏切分（`time_group_strict`）与标准化特征工程。
4. `agent_b_train`
   训练 SVM、XGBoost、MLP，并记录训练日志、指标与最佳模型。
5. `agent_c_evaluate`
   生成混淆矩阵、ROC、模型对比图；输出 JSON/Markdown 报告；执行时序泄漏检测。

## 4. Prompt 设计
LLM Planner 的核心 Prompt 固定在代码中（`code/multi_agent_langgraph_pipeline.py`）：

### 4.1 System Prompt
```text
You are an ML preprocessing planner. Return ONLY valid JSON with keys:
split, categorical, impute, scaling, feature_select, pca.
Do not include markdown or prose.
```

### 4.2 User Prompt 模板
```text
Target={target_column}, time_column={time_column}, profile={data_profile_json}.
Need robust preprocessing for classification with no temporal leakage.
```

### 4.3 输出约束
- 必须是可解析 JSON。
- 顶层键仅允许：`split, categorical, impute, scaling, feature_select, pca`。
- 计划需通过本地规则校验，失败则自动回退默认规则计划。

## 5. 异常处理机制
核心设计如下：

1. 节点级 `try/except`：单节点失败不导致整条流水线直接崩溃。
2. 统一错误记录：错误进入 `state["errors"]`，并带时间戳写入运行日志。
3. 上游失败保护：
   - Agent B 训练前校验 `X_train/X_test/y_train/y_test`；
   - Agent C 评估前校验 `training_results`。
4. 多级回退机制：
   - LLM Planner 不可用/预检失败：回退 Rule Planner；
   - XGBoost GPU 失败：回退 XGBoost CPU；
   - XGBoost 不可用：回退 `tree_fallback`。
5. 可审计输出：即使局部失败，仍尽量产出 `result.json` 与 `report.md`。

## 5.1 Imputer 策略降级
在引入 LLM Planner 后，模型可能返回不符合 `SimpleImputer` 约束的策略（例如 `bayesian_ridge`）。
为保证流水线稳定性，Agent A 增加了“白名单校验 + 自动降级”：

1. 白名单：`mean / median / most_frequent / constant`
2. 非法值处理：记录日志并自动回退为 `median`
3. 效果：避免因单个 planner 字段异常导致整条流水线中断
